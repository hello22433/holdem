## 기술적 특징 (Technical Highlights)

### 1. 동시성 제어

현재 서버는 **단일 서버 구조**이며, 빠른 개발과 낮은 인프라 비용을 목표로 **Java 고유의 모니터 락(`synchronized`)**을 사용하여 동시성을 제어했습니다.
추후 스케일 아웃이 필요할 경우, **Redis 분산 락**으로 전환할 수 있도록 설계를 염두에 두었습니다.

---

### 2. 애플리케이션 레벨 트랜잭션

메모리 기반 게임 서버에서는 DB Rollback 기능을 사용할 수 없기 때문에 **Application Level Transaction**을 구현했습니다.

* **Atomicity (원자성)**: 칩 차감과 팟 증가는 동시에 성공하거나, 실패 시 둘 다 실행되지 않도록 예외 발생 시 이전 상태(Snapshot)로 복구하는 로직을 구현했습니다.
* **Consistency (일관성)**: 플레이어 감소분과 팟 증가분이 항상 동일하도록 코드 레벨에서 검증하여, 로직 오류로 인한 자산 증발을 방지했습니다.

---

### 3. 네트워크 최적화

실시간 게임에서는 **패킷 크기 = 네트워크 비용(Latency & Bandwidth)** 입니다.

편리한 JSON 직렬화를 사용할 경우, 매 요청마다 `"playerId"`, `"action"` 같은 메타데이터가 반복 전송되어 비효율이 발생했습니다.
이를 해결하기 위해 **ByteBuffer 기반 커스텀 바이너리 직렬화**를 구현, 패킷 크기를 약 **52% 절감**하며 데이터 정합성을 유지했습니다.
결과적으로, 동시 접속자가 몰릴 때 서버 부하를 크게 줄이는 핵심 최적화가 되었습니다.
